## Layer 1: GPS Geofence (easiest, catches lazy fraud)

```typescript
// Check if capture location is within project boundary
function checkGeofence(captureGPS: GPS, project: Project): CheckResult {
  const distance = haversineDistance(captureGPS, project.coordinates);
  return {
    name: 'geofence',
    passed: distance <= project.geofenceRadiusMeters,
    score: Math.max(0, 1 - (distance / project.geofenceRadiusMeters)),
    details: { distance, threshold: project.geofenceRadiusMeters }
  };
}
```

**What it catches:** Inspector at the wrong site, or at home submitting old footage.
**What it misses:** GPS spoofing apps (addressed in V2 with cell tower cross-check).

## Layer 2: Human Tremor Detection (cheap, surprisingly effective)

Real human hands tremor at 1-3 Hz with small amplitude. A phone on a tripod or rig doesn't.

```typescript
function checkTremor(sensorLog: SensorReading[]): CheckResult {
  // Extract accelerometer magnitude over time
  const magnitudes = sensorLog.map(s => Math.sqrt(s.ax**2 + s.ay**2 + s.az**2));

  // Compute variance in a sliding 500ms window
  const windowSize = 30; // ~500ms at 60Hz
  const variances: number[] = [];
  for (let i = windowSize; i < magnitudes.length; i++) {
    const window = magnitudes.slice(i - windowSize, i);
    const mean = window.reduce((a, b) => a + b) / window.length;
    const variance = window.reduce((a, b) => a + (b - mean) ** 2, 0) / window.length;
    variances.push(variance);
  }

  const avgVariance = variances.reduce((a, b) => a + b) / variances.length;

  // Human tremor: variance between 0.001 and 0.5 m/s²
  // Tripod/rig: variance < 0.0005
  // Shaking intentionally: variance > 1.0
  const passed = avgVariance > 0.001 && avgVariance < 1.0;

  return {
    name: 'human_tremor',
    passed,
    score: passed ? 1.0 : 0.0,
    details: { avgVariance, threshold: { min: 0.001, max: 1.0 } }
  };
}
```

**What it catches:** Phone on tripod filming a TV, phone on a rig, device farm.
**What it misses:** Attacker holding the phone by hand while filming a screen (Layer 3 catches this).

## Layer 3: Sensor-Camera Consistency (the "screen replay" killer)

If the gyroscope says "device is stationary" but the video shows motion (optical flow), the camera is looking at a moving screen.

```typescript
// Server-side: extract optical flow magnitude from video frames using sharp + canvas
// Compare against gyroscope angular velocity

function checkSensorCameraConsistency(
  sensorLog: SensorReading[],
  videoFrameMotion: number[]  // optical flow magnitude per frame
): CheckResult {
  // Compute gyroscope energy (sum of angular velocity magnitude)
  const gyroEnergy = sensorLog.reduce((sum, s) =>
    sum + Math.sqrt(s.gx**2 + s.gy**2 + s.gz**2), 0) / sensorLog.length;

  // Compute visual motion energy
  const visualEnergy = videoFrameMotion.reduce((a, b) => a + b) / videoFrameMotion.length;

  // Correlation: both should be high (real movement) or both low (stationary)
  // Mismatch: visual high + gyro low = screen replay
  const ratio = visualEnergy > 0.01 ? gyroEnergy / visualEnergy : 1.0;

  // Healthy ratio: 0.3 to 3.0 (they correlate roughly)
  // Screen replay: ratio < 0.1 (lots of visual motion, no device motion)
  const passed = ratio > 0.1;

  return {
    name: 'sensor_camera_consistency',
    passed,
    score: Math.min(1.0, ratio / 0.3),
    details: { gyroEnergy, visualEnergy, ratio }
  };
}
```

**Optical flow extraction on server (Node.js):**
Use `sharp` to extract frames from uploaded video at 2fps. Compare consecutive frames pixel-by-pixel (sum of absolute differences). This gives a rough but fast motion estimate without needing OpenCV in Node.

```typescript
import sharp from 'sharp';

async function extractFrameMotion(videoBuffer: Buffer): Promise<number[]> {
  // Use ffmpeg to extract frames (via fluent-ffmpeg)
  // Then compare consecutive frames with sharp
  const frames = await extractFramesFromVideo(videoBuffer, { fps: 2 });
  const motions: number[] = [];

  for (let i = 1; i < frames.length; i++) {
    const prev = await sharp(frames[i - 1]).raw().toBuffer();
    const curr = await sharp(frames[i]).raw().toBuffer();
    let diff = 0;
    for (let j = 0; j < prev.length; j++) {
      diff += Math.abs(prev[j] - curr[j]);
    }
    motions.push(diff / prev.length); // average pixel difference
  }
  return motions;
}
```

**What it catches:** The exact demo "wow moment" — screen recording of a walkthrough, played back to the camera.
**What it misses:** Very sophisticated attacks where the attacker walks with phone in hand past a screen (tremor matches, but location doesn't — Layer 1 catches this).

## Layer 4: Perceptual Hash Duplicate Detection

Catches the developer who submits the same photo twice for different milestones.

```typescript
import { imagehash } from 'image-hash'; // or use sharp + custom dHash

async function checkDuplicate(
  newMediaHash: string,
  projectId: string,
  supabase: SupabaseClient
): Promise<CheckResult> {
  // Query all existing phashes for this project
  const { data: existing } = await supabase
    .from('media')
    .select('id, phash, created_at')
    .eq('workflow_id', projectId)
    .not('phash', 'is', null);

  // Compare hamming distance
  const THRESHOLD = 8; // out of 64 bits — very similar images
  const duplicates = existing?.filter(m =>
    hammingDistance(newMediaHash, m.phash) <= THRESHOLD
  ) ?? [];

  return {
    name: 'duplicate_detection',
    passed: duplicates.length === 0,
    score: duplicates.length === 0 ? 1.0 : 0.0,
    details: { duplicatesFound: duplicates.length, matches: duplicates.map(d => d.id) }
  };
}
```

**What it catches:** Same room photographed twice, slightly cropped or rotated versions.
**What it misses:** AI-generated synthetic images (V3 with CLIP embeddings).

## Layer 5: Challenge Code Verification

The server issued a 4-character code. The inspector wrote it on paper and held it up in the first photo. We verify it's present.

```typescript
// V1: Manual verification by supervisor (they know the code, they see the photo)
// V2: OCR via Tesseract.js to auto-detect the code in the image

function checkChallenge(workflow: Workflow): CheckResult {
  // In V1, this is a boolean set by the supervisor during review
  const challengeVerified = workflow.meta?.challenge_verified === true;

  return {
    name: 'challenge_code',
    passed: challengeVerified,
    score: challengeVerified ? 1.0 : 0.5, // 0.5 = pending supervisor check
    details: { status: challengeVerified ? 'verified' : 'pending_review' }
  };
}
```

**Why manual in V1:** OCR on construction-site handwriting is unreliable. A supervisor checking a 4-character code takes 5 seconds. Ship manual, automate in V2.

## Aggregate scoring

```typescript
const WEIGHTS = {
  geofence: 0.25,
  human_tremor: 0.20,
  sensor_camera_consistency: 0.25,
  duplicate_detection: 0.15,
  challenge_code: 0.15,
};

function aggregateScore(checks: CheckResult[]): number {
  return checks.reduce((sum, check) =>
    sum + (WEIGHTS[check.name] ?? 0) * check.score, 0
  );
}

// Threshold: >= 0.7 = auto-approve, < 0.7 = flag for human review
// Any single check with passed=false = always flag regardless of aggregate
```

# PART 4: SUPABASE REALTIME SETUP

```typescript
// In the dashboard component
import { useEffect, useState } from 'react';
import { createClient } from '@supabase/supabase-js';

function useLiveEvents(workflowId: string) {
  const [events, setEvents] = useState<LedgerEvent[]>([]);
  const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY);

  useEffect(() => {
    // Initial fetch
    supabase
      .from('ledger_events')
      .select('*')
      .eq('workflow_id', workflowId)
      .order('created_at', { ascending: true })
      .then(({ data }) => setEvents(data ?? []));

    // Live subscription
    const channel = supabase
      .channel(`workflow-${workflowId}`)
      .on('postgres_changes', {
        event: 'INSERT',
        schema: 'public',
        table: 'ledger_events',
        filter: `workflow_id=eq.${workflowId}`,
      }, (payload) => {
        setEvents(prev => [...prev, payload.new as LedgerEvent]);
      })
      .subscribe();

    return () => { supabase.removeChannel(channel); };
  }, [workflowId]);

  return events;
}
```

Enable in Supabase Dashboard → Database → Replication:
- Turn on realtime for `ledger_events` table
- Turn on realtime for `workflows` table (for state change updates)
